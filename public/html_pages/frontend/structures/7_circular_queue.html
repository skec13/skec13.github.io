Circular Queue
Circular Queue implementation
Syntax
-> The size of the queue is fixed and a single block of memory is used as if the first
element is connected to the last element
-> follows FIFO principle
-> a circular queue will reuse the empty block during the dequeue operation
-> Enqueue operation: adds an element to the rear of the collection
-> Dequeue operation: removes an element from the front of the collection
