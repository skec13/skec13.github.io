Queue implementation
Syntax
-> The queue data structure is a sequential collection of elements
that follows the principle of FIFO
-> the first element inserted into the queue is the first element to be removed
-> Enqueue operation: adds an element to the rear of the collection
-> Dequeue operation: removes the element from the front of the collection
Code
class Queue {
constructor(){
    this.items = [];
}

enqueue(element){
    this.items.push(element);
}

dequeue(){
    return this.items.shift();
}

isEmpty(){
    return this.items.length === 0;
}

peek(){
    if(!this.isEmpty()){
        return this.items[0]
    }
    return null;
}

size(){
    return this.items.length;
}

print(){
    console.log(this.items.toString());
}
}

const queue = new Queue()
queue.enqueue(10);   //adds 10
queue.enqueue(20);   //adds 20 to the end of the queue
queue.print();   //10,20
queue.dequeue();   //removes 10 from the queue
queue.print();   //20

Queue implementation optimization
Syntax
-> Shift method has linear time complexity
-> if we keep track of the current rear and front item we can make an constant time complexity
Code

class Queue {
    constructor(){
        this.items = {};
        this.rear = 0;
        this.front = 0;
    }

    enqueue(element){
        this.items[this.rear] = element;
        this.rear ++;
    }

    dequeue(){
        const item = this.items[this.front];
        delete this.items[this.front];
        this.front++;
        return item;
    }

    isEmpty(){
        return this.rear - this.front === 0;
    }

    peek(){
        return this.items[this.front];
    }

    size(){
        return this.rear - this.front;
    }

}
